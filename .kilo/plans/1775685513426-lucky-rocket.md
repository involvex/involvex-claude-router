# Plan: Fix Tunnel Connection Issues

## User Context

- Cloudflare Worker: `https://involvexs-claude-router.involvex.workers.dev`
- This is a FREE worker (not the default `tunnel.9router.com`)
- **Critical Issue**: Worker returns `{"error":"Not Found","path":"/api/session/create"}` - tunnel endpoints NOT IMPLEMENTED

## Root Cause

The tunnel API endpoints aren't implemented in the user's Cloudflare Worker. The local 9Router tries to call:

- `POST /api/session/create`
- `POST /api/tunnel/create`
- etc.

But these return 404 because `cloud/src/handlers/tunnel.js` isn't deployed to the worker's `index.js`.

## Implementation Plan

### Step 1: Add Tunnel Handlers to User's Worker

The user needs to deploy `cloud/src/handlers/tunnel.js` to their worker. They need to:

1. **Copy tunnel.js handler** from this repo to their worker:
   - File: `cloud/src/handlers/tunnel.js`
   - Contains: `handleTunnelSession`, `handleTunnelProvision`, `handleTunnelProxy`, `handleTunnelRegister`

2. **Add routes to worker's index.js**:

   ```javascript
   import {
     handleTunnelSession,
     handleTunnelProvision,
     handleTunnelRegister,
   } from "./handlers/tunnel.js";

   // Add routes
   router.post("/api/session/create", handleTunnelSession);
   router.post("/api/tunnel/create", handleTunnelProvision);
   router.delete("/api/tunnel/delete", handleTunnelProvision);
   router.post("/api/tunnel/register", handleTunnelRegister);
   ```

3. **Deploy to Cloudflare**:
   ```bash
   cd cloud
   wrangler deploy
   ```

### Step 2: Configure Local 9Router to Use Correct Worker URL

After worker is updated, set in `.env`:

```
TUNNEL_WORKER_URL=https://involvexs-claude-router.involvex.workers.dev
```

### Step 3: Improve Local Tunnel Code (Optional but Recommended)

Once worker works, improve the local code:

- Reduce connection threshold from 4→1
- Add retry logic with backoff
- Add health monitoring
